Consider a graph where cubes are vertices and edges are drawn between pairs of adjacent cubes. In this case, the surface area will be 6×5=306\times 5=306×5=30 minus the number of edges ×2\times 2×2.

It is clear that the number of edges is at least 444, and it can be shown that it cannot be 666 or more as follows.

  • Initially, there is one cube. The operation of perfectly adding cubes is repeated 444 times.
    • Up to the second operation, at most 111 edge can be added.
    • In the third and fourth operations, at most 222 edges can be added. Moreover, it is only possible to add 222 edges in either the third or fourth operation, but not both.
    • By doing this, the number of edges is at most 555.

The necessary and sufficient condition for the number of edges to be 555 is to include a cycle (which is referred to as a "namori graph" in competitive programming dialect), and it can be understood that this occurs only when the pentacube contains a shape of 2×2×12\times 2\times 12×2×1.
