サイコロを下向きに 111 回転がす移動を AAA，右向きに 111 回転がす操作を BBB とする．移動全体は A,BA,BA,B それぞれ 121212 回ずつの組み合わせからなる．最初に 666 が上を向くのは，早くて 222 回目である．また，666 が上を向いてから次に 666 が上を向くまでに，移動は最低 444 回必要とする．よって，スコアはたかだか 666 であることがわかる．
　さて，666 が上に向いてから次に 666 が上に向くまでに，必ず 111 は上に向く．また，666 が真上を向いている状態で A/BA/BA/B を実行すれば，次に A/BA/BA/B を実行したとき，またそのときに限り 111 が上を向く．これらは逆も同様である．これらのことに留意して，スコアが 666 となる移動方法を数え上げよう．スコアが 666 であるとき，AAAAAA および BBBBBB の「塊」が少なくとも 111111 個必要であるから，一方が 666 個でもう一方が 555 個である（ただし，「塊」が12個の場合には，「先頭」からの11個を考えるものとする）．BBBBBB が 666 個であるとしよう．
　このとき，AAAAAA が 555 つと BBBBBB が 666 つを 111 列に並べる方法は 11C5=462{}_{11}\mathrm{ C }_5=46211​C5​=462 通りある．それぞれについて，AAA を 222 つ適切に挿入すればよいが，適切な位置は「BBB と BBB の間」または「末尾」の 777 箇所である（「塊」の間に 222 つまとめて挿入することも可能であるが，これは AAAAAA が 666 個の場合と重複するため除外する）．よって，挿入の方法は a1+a2+⋯+a7=2a_1+a_2+\cdots+a_7=2a1​+a2​+⋯+a7​=2 なる非負整数の組 (a1,a2,…,a7)(a_1,a_2,\ldots,a_7)(a1​,a2​,…,a7​) の総数に対応し，8C2=28{}_{8}\mathrm{ C }_2=288​C2​=28 通りである．
